# 🚀 GCP Vertex AI 폐쇄망/업무망 연동 완벽 가이드
**(대상: 인프라/보안 담당자 및 시스템 관리자)**

국정자원(NIRS)과 같은 인터넷 단절(Air-gapped) 환경이나 사내 폐쇄망에서 구글의 막강한 대형 언어 모델(Gemini)을 "Vertex AI" 엔터프라이즈 모드로 연결하기 위한 표준 절차서입니다.

---

## 1. Google Cloud 환경 준비
Vertex AI 호출 권한을 얻으려면 GCP 프로젝트와 "서비스 계정(Service Account)" 키 인증서 모듈(JSON)이 필수입니다.

### 1-1. GCP 인프라 프로비저닝
1. **Google Cloud Console 접속** (조직 계정 권장)
2. 새로운 **프로젝트 생성** (예: `nurinamu-acrc-rag-ai`)
3. 프로젝트 상단 검색바에서 `Vertex AI API`를 검색하여 **사용 설정(Enable)** 클릭.
   - ⚠️ 결제 계정(Billing)이 연결되어 있어야 API가 활성화됩니다.
   - 요금은 사용한 토큰 양에 따라 월 단위 결제됩니다.

### 1-2. 서비스 계정 JSON 인증키 발급 (핵심)
누리나무 서버 코드가 구글에 인증하기 위한 파일입니다.
1. GCP 콘솔 좌측 메뉴에서 **[IAM 및 관리자]** -> **[서비스 계정]** 이동.
2. 상단 **[+ 서비스 계정 만들기]** 클릭 (이름 예: `nurinamu-vertex-caller`)
3. **[역할 부여]**: 검색창에 `Vertex AI 사용자` (또는 `Vertex AI User`)를 검색하여 정확히 부여 후 저장.
4. 생성된 계정을 클릭 후, 상단의 **[키(Keys)]** 탭 이동.
5. **[키 추가]** -> **[새 키 만들기]** -> 형식은 **JSON**으로 설정하고 다운로드.
6. 다운로드된 `xxx-12345.json` 파일은 유출을 금하며, 이후 **누리나무 관리자 화면에서 업로드하는 데 사용**합니다.

---

## 2. 보안 통신 라우팅 (폐쇄망/VPN 개방)
RHEL 7.8 등 보안 망의 서버에서 `asia-northeast3-aiplatform.googleapis.com` (GCP 서울 리전 Vertex 엔드포인트)에 도달하기 위한 보안 셋업입니다.

### [옵션 A] 클라우드 전용선 (Cloud Interconnect) / VPN 사용 - 🌟권장
국가망과 GCP 간에 직접 맞닿은 파이프(전용선)를 뚫는 방식입니다.
- **장점**: 대외 인터넷(Public Internet) 망을 전혀 거치지 않으므로 보안 심사(보안성 검토) 시 가장 확실한 논리적 근거가 됩니다.
- VPC Service Controls(VPC-SC)를 적용해 GCP 내에서도 데이터를 이중으로 가둬둘 수 있습니다.

### [옵션 B] 아웃바운드 프록시 (Proxy)를 통한 포트 개방
전용선 공사가 어렵다면 사내 중앙 인터넷 프록시 서버를 경유하도록 허가받아야 합니다.
- **방화벽 개방 대상(목적지 도메인)**: `*.aiplatform.googleapis.com`
- **프로토콜/포트**: `HTTPS (443)`
- **서버단 조치**: NIRS 장비의 설정 파일에 아래와 같이 등록하여 나갈 수 있도록 셋팅합니다.
  ```bash
  export HTTPS_PROXY="http://<프록시IP>:<포트>"
  export HTTP_PROXY="http://<프록시IP>:<포트>"
  ```

## 3. 누리나무(NuriNamu) 배포 적용

모든 사전 준비가 완료되었다면 체계에 붙이는 과정은 클릭 몇 번으로 끝납니다.

1. 누리나무 시스템을 구동 후 **관리자 화면(Admin Console)**에 로그인합니다 (`http://서버-IP:8502`).
2. 좌측 메뉴에서 **[시스템 환경설정]** 메뉴로 진입합니다.
3. 상단 **[기본 설정]**의 LLM 제공사 목록에서 **`Google Cloud Vertex AI (엔터프라이즈 GCP 환경)`**을 선택합니다.
4. 화면 중간에 위치한 API 인증 정보란에서 아래를 설정합니다:
   - **GCP Project ID 명**: 생성하신 GCP 프로젝트의 고유 ID (예: `nurinamu-acrc-rag-ai`)를 정확히 기입.
   - **GCP Location**: `asia-northeast3` (변경하지 않는 한 서울 리전 권장)
   - **JSON 인증키 업로드**: 1단계에서 다운로드 받은 `.json` 파일을 끌어다 놓습니다.
5. **[적용]**을 누르고, 하단의 **`[현재 LLM 네트워크 테스트하기]`** 버튼을 눌러 초록색 '정상 통신' 마크가 뜨면 **모든 구축이 마무리**된 것입니다.
