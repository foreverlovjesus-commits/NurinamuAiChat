# [보고서] 지능형 대국민 민원 안내 챗봇 시스템 구축 및 운영 안

**작성일:** 2026년 3월 14일
**수신:** 국민권익위원회 정보화담당관실
**발신:** 지능형 RAG 시스템 개발팀

---

## 1. 프로젝트 개요
본 프로젝트는 국민권익위원회 대표홈페이지의 대국민 서비스를 고도화하기 위해, 최신 오픈소스 거대 언어 모델(LLM)과 검색 증강 생성(RAG) 기술을 결합한 지능형 민원 응대 시스템 구축을 목표로 함.

- **예상 규모:** 일일 접속자 10,000명 / 동시 접속자 100명
- **핵심 가치:** 데이터 주권 확보(On-Premise), 표 데이터 인식 정밀화, 지능형 민원 라우팅

---

## 2. 하드웨어 아키텍처 (Hardware 구성)
고부하 환경에서의 안정적인 추론을 위해 3-Tier 이중화 구성을 제안함.

### [Option 1] 하이엔드 고성능형 (권장)
*   **WEB/WAS 서버 (2대):** x86 Standard (32Core, RAM 64GB) - Nginx 및 API 서버 가동
*   **GPU 추론 서버 (2대):** **NVIDIA H100 80GB x 8개** (NVLink)
    *   *사유:* GPT-OSS 120B 모델의 실시간 병렬 추론을 위한 충분한 VRAM 확보
*   **DB/검색 서버 (2대):** NVMe SSD 탑재 (RAM 128GB) - Qdrant, OpenSearch 가동

### [Option 2] 가성비 효율형 (양자화 모델)
*   **WEB/WAS 서버 (2대):** x86 Standard Server
*   **GPU 추론 서버 (2대):** **NVIDIA A100 80GB x 4개** 또는 **L40S 48GB x 8개**
    *   *사유:* 4-bit/8-bit 양자화 기술을 적용하여 인프라 비용 최적화
*   **DB/검색 서버 (2대):** x86 Standard Server

## 4. 업무 흐름도 (Workflow)
1.  **질의 유입:** 민원인이 홈페이지 챗봇에 자연어로 질문 입력.
2.  **메시지 큐잉:** RabbitMQ가 요청을 수집하여 가용 추론 노드로 분배.
3.  **지능형 라우팅:** Gemma 27B가 질문을 분석하여 5대 민원 창구 분류 및 처리 경로 결정.
4.  **하이브리드 검색:** OpenSearch(법령)와 Qdrant(사례)에서 관련 지식 동시 추출.
5.  **심층 추론:** GPT-OSS 120B가 추출된 지식을 바탕으로 공신력 있는 답변 생성.
6.  **가드레일 검증:** LangFuse 및 내부 가드레일을 통해 환각 현상 유무 최종 점검.
7.  **답변 전달:** Nginx를 통해 스트리밍 방식으로 답변 표출.

## 5. 프로그램 사양서 (Module Specification)

### 5.1. 지능형 인덱서 (`rag_indexer.py`)
- **기능:** PDF, HWPX, Excel 등 다중 포맷의 구조적 지식화.
- **특징:** LlamaParse 연동을 통한 표(Table) 데이터 보존, 해시 기반 증분 업데이트.

### 5.2. 비동기 리트리버 (`advanced_retriever.py`)
- **기능:** 쿼리 확장 및 멀티 소스 병렬 검색.
- **특징:** RRF(Reciprocal Rank Fusion) 알고리즘 적용, DB 커넥션 풀링 최적화.

### 5.3. RAG 메인 엔진 (`rag_engine.py`)
- **기능:** Dual-LLM 기반의 멀티 스테이지 추론 및 스트리밍 답변.
- **특징:** 민원 가이드형 UX(분류 사유 포함), SQL 보안 가드레일 탑재.
